# ProjectRift v0.5.6 test record

Date: 2026-07-14  
Branch: `main`  
Base commit: `412bc05`

## Implemented scope

- Added the Development-only `ProjectRiftSmokeTests` plugin, staged native controller, and project-level `ProjectRift.LocalSmoke` AutomationTool test node.
- Added fast content, isolated-profile, and Gauntlet-contract tests under `ProjectRift.Smoke`.
- Added fail-closed run-GUID and `-userdir` validation. Runtime profile files are derived only beneath `Saved/Automation/Gauntlet/<RunId>/Profiles`; invalid initialization does not construct a save store or scan the normal profile catalog.
- Added the PowerShell 5.1 Quick, Full, and Build pipeline, exact ProjectA editor ownership/close/reopen behavior, fixed exit codes, structured reports, and package metadata.
- Added bounded candidate directories, containment/reparse checks, Development Gauntlet gating, Shipping module exclusion, NullRHI launch probing, and rollback-safe latest-package promotion.
- Added a shared-engine manifest guard: each run validates and hash-backs up `UnrealEditor.modules` inside ProjectA, then rechecks it at every native-tool boundary without automatically modifying the shared UE installation.
- Updated the project version to 0.5.6. The persisted player profile schema remains v4.

No dual-client Gauntlet, CI service, performance benchmark, input robot, new content asset, remote profile access, Steam Cloud, or gameplay feature was added.

A new test failed at 1/29; discovery now applies the canonical ProjectA `.uproject` path in the server-side WMI query and then performs exact token validation, so unrelated editor processes do not enter the result set.

## Remaining risks

- The Gauntlet loop is intentionally single-client; it does not replace future dual-client session automation.
- Shipping commonly compiles out runtime logs and console commands, so its automated probe verifies a bounded healthy process lifetime, not an in-game map marker. Manual graphical startup remains part of acceptance.
- RunUnreal manages its device `-userdir` beneath the UE Gauntlet cache. ProjectRift profile persistence itself remains under the ProjectA automation root and is keyed by the run GUID.
- UAT continues to report unavailable SDKs for non-Win64 platforms during platform validation; Win64 is valid and both packages succeed.
