# ProjectRift v0.5.5 test record

Date: 2026-07-13  
Branch: `main`  
Base commit: `f902234`

## Implemented scope

- Added `UPRDiagnosticsSubsystem`, a Development-only 500-entry structured event ring buffer with severity, category, context, filtering, clearing, snapshot health, and validated UTF-8 JSON export.
- Added environment, local player/profile, team/session, mission, Rift, and settlement snapshot data without reading another player's profile files.
- Added `UPRDeveloperToolsSubsystem` actions for profile lifecycle, checkpoint saving, v1 sample generation, repair acceptance preparation, test loot, ready/start flow, and diagnostic export.
- Restricted primary-file corruption and one-shot save failure injection to explicitly isolated roots beneath ProjectA `Saved/Automation`; default player saves cannot enable these actions.
- Added the native `UPRDiagnosticsWidget` with Overview, Player, Team & Rift, Events, and Tools tabs. F1 toggles the panel, Escape closes it, and the main menu opens Tools when no profile is active.
- Removed automatic creation of the overlapping legacy profile, ready, and GAS debug overlays while retaining their classes for compatibility.
- Added dedicated Flow, Gameplay, Network, Save, Assets, UI, and Diagnostics categories plus explicit structured events at save, session, binding, ready, travel, Rift phase, and failure boundaries.
- Updated the project version to 0.5.5. The persisted player profile schema remains v4.

No remote log aggregation, Output Log interception, account authentication, remote profile access, new content asset, or Shipping diagnostics UI was added.

## Remaining risks

- Structured diagnostics are process-local and intentionally do not aggregate events from another machine or client process.
- Fault injection is volatile and Development-only; a process crash before an intentionally failed save cannot be retried automatically by the diagnostics system.
- Keyboard lifecycle has contract coverage and PIE lifecycle verification, but final validation should include the user's normal keyboard/input environment.
