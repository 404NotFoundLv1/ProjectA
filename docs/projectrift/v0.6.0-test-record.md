# ProjectRift v0.6.0 test record

Date: 2026-07-14  
Branch: `main`  
Base commit: `1f2ae68`

## Implemented scope

- Unified player and enemy damage under `UPRDamageExecutionCalculation` and the existing `Data.Damage` contract.
- Added physical and pollution damage typing, AttackPower scaling, pollution resistance, shield-first absorption, and invalid-input rejection.
- Added declaration-driven pollution, slow, and stun effects with replace-and-refresh semantics.
- Migrated `APREnemyCharacter` to a self-owned Minimal ASC and shared AttributeSet while retaining its public Blueprint query and attack methods.
- Added `UGA_EnemyMeleeAttack`; player primary, assault charge, and assault blast now use the shared combat-effect library.
- Added player-friendly damage/debuff rejection, inactive-target rejection, downed/dead cleanup, MoveSpeed synchronization, stun cancellation/blocking, and text debug feedback.
- Updated the displayed project version to 0.6.0. The persisted player profile schema remains v4.

No content asset, save migration, GameplayCue, production icon, VFX, audio, crit, armor, penetration, immunity, random proc, or status stacking was added.

## TDD and failure evidence

- Damage tests first failed on the missing Execution class, missing damage type tags, absent settings, and unscaled formulas, then passed after the unified execution was installed.
- Status contract tests first failed on missing native effect classes, shared library, stunned activation block, and ability definitions.
- Enemy tests first failed on the missing AbilitySystem interface/ASC/AttributeSet and replicated legacy Health/dead authority.
- Runtime status tests exposed test-frame timer behavior, player movement synchronization gaps, and a five-second periodic boundary double callback; the final frame simulation and periodic tolerance prove exactly five pollution ticks.
- Primary and assault tests first failed because player friendly fire remained and Q/E definitions were not applied to GAS enemies.
- UI and version tests first failed on the old debug layout/interfaces and the 0.5.6 project version.
