# ProjectRift v0.5.1 test record

Date: 2026-07-11  
Branch: `main`  
Base commit: `522885076b94ffdc92384582b42d28c88aa34422`

## Implemented scope

- Registered all 15 current ProjectRift GameplayTags as native C++ tags in one exported namespace.
- Removed duplicate GameplayTag INI declarations and migrated production/test consumers to the native constants.
- Added `UPRProjectSettings` as the unique Game-config source for 11 approved Vertical Slice values.
- Migrated Rift GameMode, objective, hold, spawn, and extraction consumers from Actor storage to project settings with runtime clamps.
- Preserved map paths, MissionId, enemy class selection, GAS combat tuning, authority/replication behavior, and the v0.5.0 AssetManager contract.
- Updated the project version to `0.5.1`.

No later-version feature, new mission/enemy/ability system, asset rename, map edit, or binary asset change was implemented.

### GameMode settings consumers

RED reports proved the old GameMode properties still existed and that initial stability, drain, failed retention, and return delay ignored non-default Settings CDO values.

GREEN: the complete `ProjectRift.Rift` family passed 7/7 after the four local values were removed and runtime reads were migrated.

### Objective, spawn, and extraction settings consumers

RED: ObjectiveSystem, SpawnManager, and ExtractionZone failed against non-default Settings CDO values and found all seven old Actor properties.

GREEN: the complete `ProjectRift.Rift` family passed 7/7 after component radii, hold timing, wave scaling/cap/interval, and extraction range were migrated. Scoped test overrides restore all settings without writing config files.

Post-review coverage also supplies negative and over-limit Settings CDO values. It verifies clamping for initial stability, drain, retention, return delay, objective radius/duration, spawn base/additional/cap/interval, and extraction radius. Each consumer logs a `LogProjectA` Error before using its code fallback if the settings CDO is unexpectedly unavailable.

### Packaging regression

The first UAT attempt exposed that Clamp metadata test calls were compiled into the non-editor Game target where `WITH_METADATA=0`. The metadata assertions were restricted to metadata-enabled builds; the Editor test still executes them. A direct ProjectA Win64 Development build and the subsequent full UAT run both succeeded.
