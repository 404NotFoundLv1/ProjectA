# ProjectRift v0.9.0 Production Standards Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ProjectA-only editor validation, production art/level standards, and an auditable asset-license registry without affecting packaged gameplay.

**Architecture:** `ProjectA` gains only a small reusable level-marker actor. A new `ProjectAEditor` module owns policy parsing, Asset Registry inspection, map contract checks, reporting, commandlet entry, and editor automation. UTF-8 policy and license JSON are the source of truth; human-facing production documentation explains the same rules.

**Tech Stack:** Unreal Engine 5.8 C++, Unreal editor modules, Asset Registry, JSON/JsonUtilities, commandlets, Editor automation, ProjectRift local PowerShell pipeline.

## Global Constraints

- Work only in `E:\MyWork\ProjectA`; never access another project, its editor, process, log, asset, or MCP service.
- Work on the existing `main` worktree. Do not create branches/worktrees or perform Git write operations.
- `ProjectAEditor` may load only in the Editor target. `ProjectA` remains the runtime-only module and Shipping must exclude editor/test modules.
- Validate `/Game/ProjectRift` and declared production maps only. Never rename, move, save, or auto-fix template, Variant, Blueprint, or accepted gameplay assets.
- Profile schema remains v9. No gameplay behavior, save format, or v0.9.1 work is in scope.
- Licensing evidence is fail-closed: unknown commercial/modification/attribution permission is an error. The tool checks declared evidence; it makes no legal determination.
- Every C++ behavior starts with a focused failing automation test and is only implemented after the red result is recorded.
